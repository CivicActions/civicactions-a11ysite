# Resources

Essential resources for building your accessibility expertise and staying current with best practices.

## Quick Start Resources

### If you only have 5 minutes...
- **WebAIM Introduction**: [WebAIM's Accessibility Principles](https://webaim.org/intro/) - Essential concepts explained clearly
- **WCAG Quick Reference**: [How to Meet WCAG](https://www.w3.org/WAI/WCAG21/quickref/) - Interactive guidelines reference

### If you have 30 minutes...
- **A11y Project Checklist**: [The A11Y Project Checklist](https://www.a11yproject.com/checklist/) - Practical implementation steps
- **Inclusive Design Principles**: [Microsoft's Inclusive Design](https://inclusive.microsoft.design/) - Design philosophy and methods

### If you have an hour...
- **WebAIM Screen Reader Survey**: [Screen Reader User Survey](https://webaim.org/projects/screenreadersurvey9/) - Real user insights
- **Government Accessibility Handbook**: [Section508.gov](https://www.section508.gov/) - Federal accessibility requirements

## Learning Paths by Role

### For Developers
- **MDN Accessibility**: [Developer-focused accessibility guide](https://developer.mozilla.org/en-US/docs/Web/Accessibility)
- **A11y Style Guide**: [Code examples and patterns](https://a11y-style-guide.com/style-guide/)
- **Accessible Components**: [Web Components accessibility examples](https://web-components.guide/)

### For Designers
- **Stark Accessibility Resources**: [Design accessibility handbook](https://www.getstark.co/library/)
- **Material Design Accessibility**: [Google's accessibility guidelines](https://material.io/design/usability/accessibility.html)
- **Inclusive Design Guidelines**: [BBC's accessibility principles](https://www.bbc.co.uk/accessibility/forproducts/)

### for Content Creators
- **Plain Language Guidelines**: [PlainLanguage.gov](https://www.plainlanguage.gov/)
- **Alternative Text Guide**: [WebAIM's Alt Text resource](https://webaim.org/articles/alt/)
- **Document Accessibility**: [Creating Accessible Documents](https://www.section508.gov/create/documents/)

### For Project Managers
- **Accessibility Requirements**: [Defining accessibility criteria](https://www.w3.org/WAI/planning/)
- **Testing Strategies**: [Accessibility evaluation approaches](https://www.w3.org/WAI/test-evaluate/)
- **Cost-Benefit Analysis**: [Business case for accessibility](https://www.w3.org/WAI/business-case/)

## Stay Current

### News and Updates
- **WebAIM Newsletter**: Monthly accessibility insights and updates
- **A11y Weekly**: Curated accessibility news and resources
- **Accessibility in Government**: Federal accessibility policy updates

### Communities and Events
- **A11y Slack**: Real-time accessibility discussions
- **Local Meetups**: Find accessibility groups in your area
- **Conferences**: CSUN, Inclusive Design 24, and regional events

### Podcasts and Videos
- **A11y Rules Podcast**: Practical accessibility discussions
- **13 Letters**: Accessibility community conversations
- **WebAIM's YouTube Channel**: Training videos and presentations

- **axe DevTools**: Comprehensive accessibility testing
- **Lighthouse**: Built-in browser accessibility audits

- **TPGi ARC Platform**: Testing and compliance management
